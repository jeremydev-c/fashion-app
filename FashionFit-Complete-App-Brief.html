1. Product summary

Fashion Fit is a cross-platform mobile application (Expo / React Native) backed by a Node.js API. Users photograph and catalog wardrobe items (Cloudinary-hosted images), receive AI-assisted categorization, build a Style DNA profile, obtain outfit recommendations for occasion/time/weather, chat with an AI style coach, plan outfits, view analytics, optionally use a social feed (posts, likes, follows, profiles), and manage account settings. Subscriptions via Paystack are implemented but can be disabled server-side. Push notifications (Expo) and daily nudges are supported.

3. Mobile application architecture

3.1 Entry & shell

  • App.tsx — QueryClientProvider, ThemeProvider, AuthProvider, boot/loading gate, optional AIMisuseWarning, onboarding flow, main tabs when authenticated.

3.4 API client

apiClient.ts — fetch-based apiRequest + axios-like apiClient; base URL from __DEV__ vs production or EXPO_PUBLIC_API_URL; attaches Authorization: Bearer from AsyncStorage; clears token on 401.

11. Recommendations engine (full pipeline — condensed)

Implementation: api/routes/recommendations.js (~2.3k lines). This is the most elaborate subsystem.

11.1 Request

GET /recommendations?userId=…&occasion&timeOfDay&weather&limit&variant&forecast…&weatherDetail…. Limit clamped 1–6. Middleware: daily rec limit.

11.2 Loads

Wardrobe, StyleDNA, UserPreferences, recent Outfits, LearningHistory → rejected/liked item id sets. Weather normalized (bands + detail flags + forecast).

11.3 Seed & cache

requestSeed = f(user, day, occasion, time, weather, variant, wardrobe signature, weather key). In-memory cache 10 min, keyed by seed + feedback count.

11.4 Candidate generation

buildDiverseOutfits: filter pool; weather-ranked items; dress-based and top×bottom combinations with shoes, outerwear, accessories; adaptive combo caps; quality floors; selectDiverse with overlap penalties.

11.5 Scoring

scoreOutfit: fashion intelligence + semantic cohesion + DNA + color harmony + prefs + learned weights + saved-outfit similarity + rotation + feedback + completeness + trend signals (TREND_SIGNALS).

11.6 Hybrid AI

If OPENAI_API_KEY: enhanceWithAI — diverse candidate subset, rich prompt (embeddings, weather rules), gpt-4o for formal/date else gpt-4o-mini, JSON pick by candidateIdx, ApiUsage logging, blended score, post-AI diversity + fill slots. Else formatResults.

11.7 Response

Items (always user-owned), score, confidence, reasons, title, description, archetype, optional completeTheLook, occasion/time/weather; hybrid may include algorithm vs AI confidence fields before strip.

User → API → Mongo + Cloudinary + OpenAI Wardrobe + DNA + prefs + learning → ctx → buildDiverseOutfits → score → diversify → [ OpenAI enhance ] → format → cache → JSON

14. Security & privacy (engineering view)

  • JWT for authenticated routes; tokens stored client-side (AsyncStorage — consider secure storage hardening for high-sensitivity deployments).
  • CORS allowlist configurable.
  • Rate limiting on API; request size limits still large for image payloads (tighten if abuse observed).
  • Secrets only in server environment — never commit .env.
  • AI misuse warning in app for responsible-use messaging.

Appendix A — Recommendations engine (exhaustive)

Source: api/routes/recommendations.js, api/utils/fashionIntelligence.js, api/utils/semanticStyleProfile.js. This appendix duplicates no other document — it is the full technical decomposition.

A.1 Entry & guards

  • GET /recommendations/test — sanity check.
  • GET / — enforceDailyRecommendations(); userId required; wardrobe min 2 items or 400; limit clamped 1–6.

A.2 Query parameters

userId, occasion (aliases→profiles: casual, work, date, party, gym, formal), timeOfDay, weather or temperature-derived band, temperature, condition, humidity, windSpeed, needsLayers, hasRainRisk, tempSwing, limit, variant (seed entropy).

A.3 Data & feedback

Parallel fetch: StyleDNA, UserPreferences, Outfits (~30), LearningHistory (~100). Feedback → rejectedItemIds / likedItemIds from reject/save/swipe/rate types.

A.4 Context & cache

ctx holds wardrobe, wardrobeProfile, wardrobeById, semanticProfileMap, DNA, prefs, saved outfits, feedback sets, normalized occasion/time/weather, weatherDetail, forecast, requestSeed. Cache: Map, TTL 10 min, key includes feedback count; evict when >500 entries.

A.5 Pool & combinations

Filter avoided colors (restore if pool too small). Preferred vs fallback via occasion profile, checkItemAppropriateness, weatherItemScore. Categories: tops, bottoms, shoes, outerwear, accessories, dresses. needsOuterwear; dress path + top×bottom loops; rankPool with pairCompat + rotation + itemPreferenceFit + seeded noise; adaptive maxCombos (30–600) and top/bottom iteration caps; accessory variants (none / best / two varied); sparse-wardrobe fallbacks; last-resort 2–4 item group.

A.6 Quality & diversity

Quality floor varies by wardrobe size; filter by score and intel.score; relax to all if too few. selectDiverse: penalties for reusing tops/bottoms/dresses/shoes (conditional), direction keys, palette/style groups, item overlap, semantic overlap; diversity weight scales with wardrobe size.

A.7 makeCandidate

evaluateFashionIntelligence once; getCandidateSemanticDescriptor; semanticOutfitCohesionCached; scoreOutfit; buildReasons; getOutfitDirection.

A.8 scoreOutfit components

Intel score weight; semantic cohesion; Style DNA primary/secondary match; outfit color harmony; preference match + avoided; learned color/style/category/pattern weights; per-occasion preference map; avoided color combos + style: avoids; saved-outfit Jaccard + semantic similarity bands; rotation + fresh core bonus; liked/rejected item presence; completeness adapted to wardrobe; item-count sweet spot; trendAlignmentScore vs TREND_SIGNALS; clamped output band.

A.9 Hybrid AI

If OPENAI_API_KEY: enhanceWithAI — user profile string; diverseForAI subset; prompt with season, weather line, alert rules, wardrobe counts, JSON candidates with embeddings; rules for distinct tops/bottoms; gpt-4o for formal+date else gpt-4o-mini; temperature ~0.82; parse JSON; ApiUsage.create; blend algo score + AI confidence; deduplicateAccessories; classifyArchetype; completeTheLook; post-AI diversity gate; fill from remaining candidates; appendFallbackResponses if needed; strip internal fields. On failure → formatResults.

A.10 formatResults (algorithm-only path)

Final diversity gate (with relaxations for tiny wardrobes); buildPresentationCopy; computeRecommendationConfidence; stable makeRecommendationId.

A.11 Test exports

router.__testables — seed helpers, makeCandidate, selectDiverse, formatResults, pairCompat, normalizers.
